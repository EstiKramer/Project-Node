API User Operations

שיטה כתובת פרמטרים ב-body מה עושה סוגי שגיאות אפשריות
GET /api/user/ אין מחזיר את כל המשתמשים
  • 500 - שגיאה בשרת
  • 400 - בעיה בביצוע הבקשה
GET /api/user/:id אין מחפש משתמש לפי ID
  • 404 - משתמש לא נמצא
  • 400 - ID לא תקני
  • 500 - שגיאה בשרת
POST /api/user/ 
{
    "username": "שם המשתמש",
    "email": "כתובת אימייל",
    "password": "סיסמה",
    "role": "תפקיד",
    "registration": "תאריך רישום"
}
יוצר משתמש חדש במערכת
  • 404 - חסר פרמטרים (שם משתמש, אימייל או סיסמה)
  • 500 - שגיאה בשרת
POST /api/user/login 
{
    "username": "שם המשתמש",
    "password": "סיסמה"
}
מפעיל כניסה למערכת
  • 404 - שם משתמש או סיסמה שגויים
  • 500 - שגיאה בשרת
PUT /api/user/:id 
{
    "username": "שם חדש",
    "email": "אימייל חדש",
    "role": "תפקיד חדש",
    "registration": "תאריך רישום חדש"
}
מעדכן את פרטי המשתמש לפי ID
  • 404 - משתמש לא נמצא
  • 400 - ID לא תקני
  • 500 - שגיאה בשרת
PUT /api/user/:id 
{
    "newPassword": "סיסמה חדשה"
}
מעדכן את הסיסמה של המשתמש לפי ID
  • 404 - משתמש לא נמצא
  • 400 - ID לא תקני
  • 400 - סיסמה לא תקינה
  • 500 - שגיאה בשרת
PUT /api/user/:id 
{
    "newPassword": "סיסמה חדשה"
}
עדכון סיסמה למשתמש לפי ID
  • 404 - משתמש לא נמצא
  • 400 - סיסמה חדשה חסרה
  • 500 - שגיאה בשרת
DELETE /api/user/:id אין מוחק משתמש לפי ID
  • 404 - משתמש לא נמצא
  • 400 - ID לא תקני
  • 500 - שגיאה בשרת
