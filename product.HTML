API Product Operations

שיטה כתובת פרמטרים ב-body מה עושה סוגי שגיאות אפשריות
GET /api/product/ אין מחזיר את כל המוצרים
  • 500 - שגיאה בשרת
  • 400 - בעיה בביצוע הבקשה
GET /api/product/:id אין מחפש מוצר לפי ID
  • 404 - מוצר לא נמצא
  • 400 - ID לא תקני
  • 500 - שגיאה בשרת
POST /api/product/ 
{
    "productName": "שם המוצר",
    "description": "תיאור המוצר",
    "DateOfManufacture": "תאריך ייצור",
    "imagePath": "כתובת תמונה",
    "price": "מחיר",
    "sizes": [
        { "size": 42, "stock": 10 },
        { "size": 44, "stock": 5 }
    ],
    "type": ["סוג 1", "סוג 2"]
}
יוצר מוצר חדש במערכת
  • 400 - שדה חסר (כגון productName או price)
  • 400 - מחיר לא תקני (לא מספר חיובי)
  • 400 - תאריך ייצור לא תקני
  • 500 - שגיאה בשרת
PUT /api/product/:id 
{
    "productName": "שם חדש",
    "description": "תיאור חדש",
    "DateOfManufacture": "תאריך ייצור חדש",
    "imagePath": "כתובת תמונה חדשה",
    "price": "מחיר חדש",
    "sizes": [
        { "size": 42, "stock": 10 },
        { "size": 44, "stock": 5 }
    ],
    "type": ["סוג חדש"]
}
מעדכן מוצר קיים לפי ID
  • 404 - מוצר לא נמצא
  • 400 - ID לא תקני
  • 500 - שגיאה בשרת
DELETE /api/product/:id אין מוחק מוצר לפי ID
  • 404 - מוצר לא נמצא
  • 400 - ID לא תקני
  • 500 - שגיאה בשרת
